

1.  MissingComponentException: There is no 'AAA' attached to the "BBB" game object, but a script is trying to access it.

遺失了AAA元件在名稱為BBB的物件上，這一般發生在當你使用GetComponent系列的語法時，他沒有找到東西。
注意GetComponent是指取得"自己"這個物件上的元件，所以確認好GetComponent到底有沒有抓到東西就可以了。

2.  UnassignedReferenceException: The variable YourVariable of YourScript has not been assigned.

這是當你定義了某個public的變數，卻忘記把東西指定給這個變數時產生的。 不需要多想，檢查一下變數的欄位有沒有東西！

3.  NullReferenceException: Object reference not set to an instance of an object

這個跟第2點有點類似，可能是因為某個變數是  "null空的" ，但這個變數不限於public變數，嘗試使用腳本中任何空的變數時，都可能會發生。
檢查一下他所指出的那一行，是不是有什麼變數忘了給東西。
另外當你在取用一個陣列元素時，如果這個元素是null空的也會出現(即使其他元素不是空的)。

4.  `UnityEngine.Application.LoadLevel(int)' is obsolete: `Use SceneManager.LoadScene'

這個是在某一次更新後就會出現，只要你在新版的專案中有使用到Application.LoadLevel語法，他就會給你一個警告。
這是說LoadLevel是不太理想的轉換場景方式，建議使用SceneManager
事實上不理他也沒關係。

5.IndexOutOfRangeException: Array index is out of range.

顧名思義，超出陣列範圍了！ 檢查一下你給陣列的index是不是確實落在陣列的有效範圍內。
如果陣列有 5 個東西，那有效範圍就是 0~4

6.Literal of type double cannot be implicitly converted to type `float'. Add suffix `f' to create a literal of this type

C#中，如果定義的常數為float類型，那必須在數字的後方加上f，否則會出現此錯誤。
public float myFloat = 1.0f

7.unexpected char: 0x8ACB.

低機率遇到的問題，跟抽到SSR的機率差不多。
這個問題是出現了隱藏字元在檔案中，只要把該行 "重寫" 就好。
注意因為是隱藏字元，所以不可以用複製的(一樣會複製到它)，一定要刪除重寫

另外也有可能是因為程式內出現不是""形式的中文字

8.SendMessage YourMessage has no receiver!

在使用GameObject.SendMessage時會發生的錯誤，這是因為他找不到符合的方法名稱
原則上這個錯誤不會對遊戲進程造成影響，但你如果覺得他很礙眼，可以在加入一個參數：

SendMessage("YourMessage") ;   原本是這樣
SendMessage("YourMessage", SendMessageOptions.DontRequireReceiver) ;  改成這樣，意思是 不需要接收對象

9.Parameter 'Attack' does not exist.

當使用Animator.SetBool  SetFloat  SetTrigger 系列的語法時，如果系統找不到符合的參數名稱就會出現此錯誤。
基本上這時候系統就會忽略他，對遊戲進程沒有影響。

10.MissingReferenceException: The object of type 'YourType' has been destroyed but you are still trying to access it.

這個問題發生在，當呼叫Destroy()某個物件或元件後，其他的腳本卻又嘗試取得他時就會報錯。

例如
Destroy(GetComponent<Collider>()) ;
GetComponent<Collider>().enabled = false ;

因為前面已經先Destroy掉，但是後面馬上又去取得他，產生這種矛盾的狀況時，系統就會報錯。
所以只要檢查"順序"有沒有出錯就好，呼叫Destroy之後就不要再讓其他東西去動到他。

11.warning CS0414: The private field `someComponent' is assigned but its value is never used
warning CS0168: The variable `someVariable' is declared but never used

這兩句類似，當定義一個變數卻沒有去使用這個變數時，就會出現這個警告。
可以不理他沒關係，但是定義變數後就會佔用記憶體空間，所以最好不要有浪費的狀況。

12.ArgumentException: Input Axis MouseY is not setup.

這個錯誤是當使用GetAxis  GetButton系列語法時，他找不到指定的Input
例如Input.GetAxis("MouseY")   然而預設在InputManager中的是"Mouse Y"，差一個空格也會有差喔！
只要到Edit > Project Settings > Input中去做設定即可

13.Unexpected symbol `}', expecting `;'

沒什麼特別，不是少了括號就是分號，或是少了 點

14.Cannot modify a value type return value of `UnityEngine.Transform.eulerAngles'. Consider storing the value in a temporary variable

C#中 沒辦法直接修改像是transform.position.x   velocity.x這樣的值，必須先用一個暫存變量將他存起來

假設想要這麼做：
transform.position.x = 4f ;

就必須額外加入兩行：
Vector3 pos = transform.positon ;
pos.x = 4f ;
transform.position = pos ;
